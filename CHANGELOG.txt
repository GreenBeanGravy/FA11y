7/1/2026 #1:
- Removed the damage reader and its PaddleOCR text-recognition dependencies. The DamageReader setting and the Speak Last Damage keybind are gone, and the installer no longer asks whether you have an NVIDIA GPU.

6/27/2026 #1:
- Fixed custom POI creation never finding the player's location. It now uses the same robust PPI pipeline as navigation, so it works whether the full-screen map is open or closed (when the map is open, it briefly closes it to read the minimap, then reopens it). The default keybind is now Left Alt + C.

6/24/2026 #1:
- Added the Check Display Mode keybind (Left Alt + R). It reads Fortnite's game log and announces the current window mode (Fullscreen, Windowed Fullscreen, or Windowed) followed by the render resolution.

6/22/2026 #1:
- New match detection now reads Fortnite's game log instead of the on-screen skydive height bar, which had become unreliable. Visited game objects now reset reliably at the start of every match.
- The skydive altitude callouts ("X meters high") now run only in Fortnite OG, where the height bar is still dependable.

6/17/2026 #1:
- Added a damage reader (off by default). When enabled, FA11y reads Fortnite's hit-damage numbers near the center of the screen and announces each engagement's total damage once the number stops counting up. Only runs during active gameplay, not in menus, the inventory, the map, or the lobby.
- Added the Speak Last Damage keybind (Left Alt + D) to repeat the most recent damage total.
- The damage reader's text recognition runs on your NVIDIA GPU if you have one (chosen during install), otherwise on the CPU.

6/15/2026 #1:
- Fixed saved Epic Games logins not persisting across restarts. FA11y now restores the session from the saved refresh token on startup instead of re-running the browser login every launch, and no longer drops the refresh token when Epic doesn't rotate it.

3/26/2026 #1:
- Added mouse passthrough DPI as a configurable setting in the configuration GUI (Values tab). Removed the console DPI prompt during first-time setup.
- Fixed configuration GUI window being too small and not properly filling out. Added minimum size and proper scrollbar support.

3/25/2026 #1:

- Updated emote wheel to new 8-slot circular layout
- Fixed equip button not working for Jam Tracks, instruments, and other new cosmetic types
- Mouse passthrough now auto-starts on launch with new keybinds (Ctrl+Shift+M to recapture, Ctrl+Shift+P to toggle)
- Significant performance improvements across screen reading, monitors, and startup time
